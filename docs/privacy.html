Privacy Policy — Emberfile

Last updated: 2026-07-02

What this extension does

Emberfile is a Chrome extension that automatically renames your downloaded files using AI-suggested names. The first time it sees a new file pattern, it sends a small set of metadata about that download (never the file itself) to a developer-operated relay to generate a suggested rename. Every repeat of that same pattern is handled entirely on your device — nothing is sent anywhere.

Your consent, and when data is sent

No data leaves your device until you explicitly accept the first-use consent screen shown the first time the extension activates. Once accepted, data is sent to the relay only on a cache miss — the first time a new file-naming pattern is seen. Every subsequent download that matches an already-learned pattern is renamed locally, with zero network activity.

How your data flows

When a new (never-before-seen) file pattern downloads, the following two-hop flow occurs:

  1. Extension → Developer relay: The extension sends the filename, MIME type, file size, and the sanitized (query-string-stripped) hostname+path of the download source and referrer to a relay endpoint operated by the developer (Trevor McAllister).
  2. Developer relay → Claude Haiku (Anthropic): The relay forwards this metadata to Anthropic's Claude Haiku API to generate a suggested rename and category tag.
  3. Result returned: Claude returns a suggested name and tag. The relay returns this to your extension, which stores a local rule so the same pattern never needs another API call. The relay logs only: the suggested tag, the rename format, and a timestamp. Your actual filename and source URLs are not retained in server logs.

API key and account

The developer holds the Claude API key. You do not need to provide or see an API key. No user account, login, or registration is required to use this extension.

No other third parties receive your data.

Data retention

  • The extension stores learned rename rules in your local browser storage (chrome.storage.local). This data never leaves your device except as described above.
  • Server logs retain only the anonymized pattern, tag, rename format, and timestamp. Logs are retained for a maximum of 30 days.
  • No personal data is retained server-side.

Your rights

You may clear all locally stored rules at any time by uninstalling the extension or by clearing the extension's storage via Chrome's developer tools.

To request deletion of any server-side log data, contact: ColdtypeLabs.support@proton.me

Changes to this policy

If this policy changes materially, the effective date above will be updated and the change will be noted in the extension's changelog.

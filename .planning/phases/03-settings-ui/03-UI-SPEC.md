## Spacing Scale

Declared values (multiples of 4 — Tailwind's default scale):

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge padding, close button padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, button icon gap |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card inner padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, modal padding |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between major sections |
| 2xl | 48px | `py-12` | Major section breaks (options page section dividers) |
| 3xl | 64px | — | Not used — extension UI is compact |

Exceptions:
- Popup fixed width: 360px (`w-[360px]`) — D-01.
- Touch targets (Pause/Resume, Save format, Discard, Yes, Upgrade): minimum 36px height (`h-9` = 36px) for click precision in a small popup.
- Modal overlay: full viewport (`fixed inset-0`); modal card: max-width 480px (`max-w-sm` or `max-w-md` depending on content).

## Typography

| Role | Size | Tailwind | Weight | Tailwind | Line Height | Notes |
|------|------|----------|--------|----------|-------------|-------|
| Body | 14px | `text-sm` | 400 regular | `font-normal` | 1.5 (`leading-relaxed`) | Default prose, rule format display, meta labels |
| Label / Button | 14px | `text-sm` | 600 semibold | `font-semibold` | 1 (`leading-none`) | All button labels, section labels, icon aria-labels |
| Heading (section) | 16px | `text-base` | 600 semibold | `font-semibold` | 1.2 (`leading-tight`) | "Learned Rules", "Custom Rules", "Account" headings |
| Display (popup header) | 18px | `text-lg` | 700 bold | `font-bold` | 1.2 (`leading-tight`) | ONLY the "Download Renamer" title in popup and options header |

Sizes declared: 4 (14, 14, 16, 18px). Weights declared: 2 (regular 400 + emphasis 600/700).

**Weight clarification:** 700 (`font-bold`) is used ONLY for the display heading "Download Renamer" in the popup header and options page header. 600 (`font-semibold`) is used for all other emphasis — section headings, button labels, badge text, and link-style CTAs. The executor must not apply `font-bold` to any element other than the display heading.

> Source: CONTEXT.md D-04 (consumer-app feel), Tailwind defaults, no custom font declared.

## Color

| Role | Tailwind Token | Hex (approx) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `bg-white` | #ffffff | Popup card background, options page background, modal background |
| Secondary (30%) | `bg-gray-50` / `bg-gray-100` | #f9fafb / #f3f4f6 | Rule row backgrounds (alternating or hover), section divider fill, slot hint area |
| Accent (10%) | `bg-blue-600` / `bg-indigo-600` | #2563eb / #4f46e5 | See reserved list below |
| Destructive | `bg-red-100` + `text-red-700` | #fee2e2 / #b91c1c | Delete confirmation row background + "Delete?" label + "Yes" button |

**Accent (`blue-600`) reserved for:**
1. Pause/Resume button (primary CTA in popup) — `bg-blue-600 text-white`
2. "Manage rules →" button in popup — `text-blue-600` (text link style, not filled)
3. "Save format" button in inline edit row — `bg-blue-600 text-white`
4. "Add custom rule" button — `bg-blue-600 text-white`
5. "Pick this" button in conflict resolution modal (both options use `bg-blue-600 text-white`)
6. "Upgrade" button in options page account section — `bg-indigo-600 text-white`
7. FREE / PREMIUM badge background — `bg-blue-100 text-blue-700` (FREE) / `bg-indigo-100 text-indigo-700` (PREMIUM)

**Text colors:**
- Primary body text: `text-gray-900`
- Secondary / meta text: `text-gray-500`
- Slot hint text: `text-gray-400 text-xs`

> Source: D-04 (white card, blue/indigo accent), D-16 (red delete row), specifics section.

## Visuals

**Primary visual anchor (popup):** The Pause/Resume button is the focal point of the popup's primary screen. It must render full-width with a filled `bg-blue-600` background, above the stats line, so the user's eye lands on it first. No other element in the popup competes for visual weight at this level.

## Component Inventory

Every UI component required for Phase 3:

### Popup (`entrypoints/popup/App.tsx`)

| Component | Description |
|-----------|-------------|
| `PopupShell` | Fixed 360px white card. Flex column, `p-4` inner padding. |
| `PopupHeader` | Row: extension icon (16×16 or 20×20) + "Download Renamer" in `text-lg font-bold text-gray-900`. |
| `PauseResumeButton` | Full-width button. Label switches between "Pause" (active) and "Resume" (paused). `bg-blue-600 text-white rounded-md h-9 font-semibold text-sm`. Renders above stats line — primary visual anchor. Reads/writes `local:enabled`. |
| `RenameCountLabel` | `text-sm text-gray-500`. Reads `local:monthlyCount`. |
| `AccountBadge` | Pill badge. FREE: `bg-blue-100 text-blue-700`. PREMIUM: `bg-indigo-100 text-indigo-700`. `text-xs font-semibold px-2 py-0.5 rounded-full`. |
| `ManageRulesLink` | Text-style button `text-blue-600 text-sm font-semibold`. On click: `chrome.tabs.create({ url: chrome.runtime.getURL('options.html') })`. |

### Options Page (`entrypoints/options/App.tsx`)

| Component | Description |
|-----------|-------------|
| `OptionsShell` | Full-tab page. `min-h-screen bg-white`. Max-width container `max-w-2xl mx-auto px-6 py-8`. |
| `OptionsHeader` | Same logo + "Download Renamer" branding as popup (D-10). `text-lg font-bold text-gray-900`. |
| `StatsSection` | Rule count + monthly rename count. `text-sm text-gray-500`. |
| `AccountSection` | Shows FREE/PREMIUM badge. Free users see `Upgrade to Premium` button (`bg-indigo-600 text-white`). Opens external URL in new tab (D-11). |
| `SectionHeading` | Reusable `<h2>` — `text-base font-semibold text-gray-900 mb-4`. Used for "Learned Rules" and "Custom Rules". |
| `RuleList` | Scrolling `<ul>` of rule rows. No search/filter (D-12). |
| `RuleRow` | Default state: displays `{tag} → {renameFormat}`. Right side: pencil icon + trash icon. `text-sm text-gray-900`. Hover: `bg-gray-50`. |
| `EditExpandedRow` | Expands below the rule row (not modal). Text input pre-filled with current `renameFormat`. Slot hint line below. "Save format" + "Discard" buttons. |
| `DeleteConfirmRow` | Replaces rule row content. `bg-red-50`. "Delete this rule?" in `text-red-700`. "Yes, delete" + "Cancel" buttons. |
| `EmptyRuleState` | Shown when rule list has 0 items. Heading + body copy per copywriting contract. `text-sm text-gray-500`. |
| `CustomRuleRow` | Similar to RuleRow but displays match condition: `contains "{matchText}" → {renameFormat}`. |
| `AddCustomRuleButton` | `bg-blue-600 text-white` button at bottom of Custom Rules section. Opens `CustomRuleModal`. |
| `CustomRuleModal` | Centered modal overlay (`fixed inset-0 bg-black/50`). Card: `bg-white rounded-lg p-6 max-w-sm w-full`. Fields: match text input + rename format input + slot hint + "Add rule" / "Discard" buttons. |
| `ConflictModal` | Shown on mount if `local:pendingConflict` is non-null. Side-by-side display of custom vs. learned `renameFormat`. Each side has "Use this" button. Overlay same as CustomRuleModal. No dismiss without picking (per D-23). |

## Interaction States

| Component | States |
|-----------|--------|
| PauseResumeButton | active (Pause, blue) → paused (Resume, blue outline or gray) |
| RuleRow | default → hover (`bg-gray-50`) → edit-expanded → delete-confirm |
| EditExpandedRow | idle → dirty (input changed) → saving (button disabled) → saved (row collapses) → error |
| DeleteConfirmRow | confirm shown → deleting (buttons disabled) → row removed → error |
| CustomRuleModal | closed → open → submitting → closed (rule added) → error |
| ConflictModal | not shown → shown on mount (if conflict pending) → user picks → cleared |

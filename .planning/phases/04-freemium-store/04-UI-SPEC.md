## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `py-1`) | Icon gaps, badge internal padding |
| sm | 8px (`gap-2`, `py-2`, `px-2`) | Inline element spacing, compact padding |
| md-compact | 12px (`gap-3`, `px-3`, `py-3`) | Internal component padding (upgrade banner, button height units) |
| md | 16px (`gap-4`, `p-4`, `mb-4`, `px-4 py-2`) | Default element spacing, section margins |
| lg | 24px (`p-6`, `gap-6`) | Container padding (options page outer) |
| xl | 32px | Layout gaps (not currently used in popup) |
| 2xl | 48px | Major section breaks |

Exceptions:
- `h-9` (36px) on buttons — established in Phase 3 for all primary/secondary buttons. Not on the 8-point scale but locked by prior phase for visual consistency with existing buttons.
- Popup total width: `w-[360px]` — arbitrary, locked in Phase 1.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|---------------|
| Brand title | 18px | 700 (bold) | 1.4 (auto) | `text-lg font-bold` |
| Section heading | 16px | 600 (semibold) | 1.4 (auto) | `text-base font-semibold` |
| Body / label | 14px | 400 (regular) | 1.5 | `text-sm` |
| Badge / secondary | 12px | 600 (semibold) | 1 (none) | `text-xs font-semibold` |

**Weights declared:** 400 (regular), 600 (semibold). The `font-bold` (700) on the brand title is a single-use brand exception, not a system weight — executors should not introduce 700-weight elsewhere.

**Constraint:** Max 4 font sizes (12, 14, 16, 18). Max 2 system weights (400, 600). Phase 4 adds no new sizes or weights.

**60/30/10 split declared:** ✓ (white/gray dominates surface area; blue+indigo together ≈10%)

## Visual Hierarchy & Focal Points

### Popup (normal state)

1. **Primary focal point:** Pause/Resume button (blue-600, full-width, h-9) — first interactive element, draws eye immediately
2. **Secondary focal point:** Count label + AccountBadge row — usage status at a glance
3. **Tertiary:** "Manage rules →" link — secondary navigation

### Popup (at-limit state, new in Phase 4)

1. **Primary focal point:** Pause/Resume button (unchanged)
2. **Secondary focal point:** Upgrade banner (amber) — amber color creates visual contrast against white surface, pulls attention below the count label
3. **Tertiary:** "Manage rules →" link

**Focal point principle:** Banner amber color is the only non-blue/white/gray element visible — it naturally becomes the secondary focus without competing with the primary CTA.

### Options Page — AccountSection (at-limit, key input expanded)

1. **Primary focal point:** Upgrade to Premium button (indigo-600)
2. **Secondary focal point:** License key input + Activate button (appears below on "Have a key?" expand)
3. **Tertiary:** "Have a key?" expand link (small, `text-sm text-gray-500 underline`) — intentionally de-emphasized

**Visual hierarchy principle:** Upgrade button is visually dominant (indigo, h-9). Key redemption is deliberately secondary to avoid drawing non-paying users into a key-entry flow.

## Component Specifications

### Upgrade Banner (popup)

```
bg-amber-50 border border-amber-200 rounded
px-3 py-2
flex items-center justify-between
```

- Left: `<span className="text-amber-800 text-xs">"Monthly limit reached"</span>`
- Right: `<button className="text-amber-800 text-xs font-semibold underline">"Upgrade to Premium"</button>`
- Click: `chrome.tabs.create({ url: UPGRADE_URL })`
- Visibility: `{!isPremium && count >= 5 && (<div>...)}`

### "Have a key?" Expand Link

```
text-sm text-gray-500 underline cursor-pointer
mt-2
```

- Renders below the "Upgrade to Premium" button
- Click: `setShowKeyInput(true)` — collapses to key input row, link disappears
- Only visible when `!isPremium && !showKeyInput`

### License Key Input Row (expanded state)

```
flex gap-2 items-center mt-2
```

- Input: `text-sm border border-gray-200 rounded-md px-3 h-9 flex-1 focus:outline-none focus:ring-2 focus:ring-indigo-500`

- Inline error: `text-xs text-red-600 mt-1` — below the input row

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Chrome download API → background.ts | downloadItem.filename is untrusted input from Chrome; used as string only |
| background.ts → Cloudflare Worker | Extension POSTs filename, mimeType, fileSize — no secrets, no URLs per D-04 |
| Worker response → background.ts | Parsed as JSON; fields used as string values only, not executed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Denial of Service | Worker fetch hang | mitigate | Promise.race with 5000ms timeout. If timeout fires, error is caught and suggest() called with no args. Download never hangs regardless of relay latency. |
| T-02-03-02 | Tampering | suggestedName used as filename | mitigate | suggestedName is used as a string value passed to suggest(). Chrome's conflictAction: 'uniquify' handles filesystem collisions. No path traversal risk — Chrome's suggest() API handles the final path construction. |
| T-02-03-03 | Information Disclosure | Download URL sent to Worker | mitigate | downloadItem.url is explicitly NOT included in the Worker POST body per D-04. Only filename, mimeType, fileSize are sent. |
| T-02-03-04 | Tampering | JSON parse of Worker response | mitigate | Worker already extracts first {…} substring. In background.ts, response.json() parse failure is caught by the catch block → suggest() called with no args → original filename used. No code path executes the returned string. |
| T-02-03-SC | Tampering | npm installs | accept | No new packages installed in this plan. wrangler and @cloudflare/workers-types already installed in Plan 02-02. |

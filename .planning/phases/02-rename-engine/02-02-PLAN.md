    Create workers/rename-relay/index.ts implementing the Cloudflare Worker relay. Key requirements:

    CORS_HEADERS constant object: Access-Control-Allow-Origin '*', Access-Control-Allow-Methods 'POST, OPTIONS', Access-Control-Allow-Headers 'Content-Type'.

    OPTIONS method → return Response(null, { headers: CORS_HEADERS }) immediately.

    Non-POST, non-OPTIONS → return Response('Method Not Allowed', { status: 405 }).

    POST handler:
    1. Parse request body as { filename: string, mimeType: string, fileSize: number }.
    2. Call fetch('https://api.anthropic.com/v1/messages', { method: 'POST', headers: { 'Content-Type': 'application/json', 'anthropic-version': '2023-06-01', 'x-api-key': env.ANTHROPIC_API_KEY }, body: JSON.stringify({ model: 'claude-haiku-4-5-20251001', max_tokens: 256, system: SYSTEM_PROMPT, messages: [{ role: 'user', content: `filename: ${body.filename}\nmimeType: ${body.mimeType}\nfileSize: ${body.fileSize}` }] }) }).
    3. If !res.ok → return Response(JSON.stringify({ error: 'upstream_error' }), { status: 502, headers: { ...CORS_HEADERS, 'Content-Type': 'application/json' } }).
    4. Parse response: data.content[0]?.text ?? '{}'.
    5. Extract first JSON object: text.match(/\{[\s\S]*\}/) — use match[0] if found, else '{}'. (Pitfall 3 mitigation.)
    6. Return Response(jsonStr, { headers: { ...CORS_HEADERS, 'Content-Type': 'application/json' } }).

    Wrap entire POST logic in try/catch — catch returns { error: 'worker_error' } with status 500.

    Export: export default { async fetch(request, env) { ... } } satisfies ExportedHandler<Env>;

    SYSTEM_PROMPT: const SYSTEM_PROMPT = the zero-shot schema description from the interfaces block above.

    Do NOT use any Node.js APIs (no Buffer, no node:https, no process). Use only Web APIs (fetch, Request, Response, JSON). This is the Workers runtime.

    Do NOT add the @cloudflare/workers-types import — add it to tsconfig via `@cloudflare/workers-types` or via `wrangler types` generation. The types are ambient.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Extension → Cloudflare Worker | Extension POSTs filename/mimeType/fileSize to open relay. No auth in Phase 2 (deferred to Phase 4). |
| Cloudflare Worker → Anthropic | Worker sends filename context with API key in header. Key never exposed to extension or user. |
| filename → Claude prompt | Filename string from Chrome download API included in Claude user message. Potential prompt injection vector. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Information Disclosure | ANTHROPIC_API_KEY | mitigate | Key stored in Cloudflare secret binding only. Never in wrangler.toml [vars], VITE_ env vars, or extension bundle. Verified by acceptance criteria (grep for sk-ant-). |
| T-02-02-02 | Elevation of Privilege | Open Worker relay | accept | Relay is open in Phase 2 (no auth) per CONTEXT.md deferred decision. Any caller can trigger an Anthropic API call at Trevor's cost. Rate limiting and auth added in Phase 4. Risk accepted: low-value target, small user base in Phase 2. |
| T-02-02-03 | Tampering | Prompt injection via filename | mitigate | Filename is sent as user message content, not interpolated into the system prompt. Claude's response is parsed as structured JSON — not executed. Worst case: Claude produces malformed JSON → JSON parse fails → extension falls back to original filename. No code execution path. |
| T-02-02-04 | Denial of Service | Worker error propagation | mitigate | Worker wraps all POST logic in try/catch returning 500. Upstream Anthropic errors return 502. Neither path throws an unhandled exception. Extension timeout (5-second Promise.race, enforced in Plan 02-03) bounds the blast radius. |
| T-02-02-SC | Tampering | npm install (wrangler, @cloudflare/workers-types) | accept | Both packages verified in Package Legitimacy Audit in RESEARCH.md — slopcheck OK, confirmed on npm registry. No [ASSUMED] or [SUS] packages. |
</threat_model>
